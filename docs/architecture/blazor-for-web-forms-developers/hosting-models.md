<span data-ttu-id="425ea-122">Blazor WebAssembly アプリは、純粋にクライアント側で実行されます。</span><span class="sxs-lookup"><span data-stu-id="425ea-122">Blazor WebAssembly apps run purely client-side.</span></span> <span data-ttu-id="425ea-123">このようなアプリは、GitHub Pages や Azure の静的 Web サイト ホスティングなどの静的サイト ホスティング ソリューションに配置できます。</span><span class="sxs-lookup"><span data-stu-id="425ea-123">Such apps can be deployed to static site hosting solutions like GitHub Pages or Azure Static Website Hosting.</span></span> <span data-ttu-id="425ea-124">サーバー上では、.NET はまったく必要ありません。</span><span class="sxs-lookup"><span data-stu-id="425ea-124">.NET isn't required on the server at all.</span></span> <span data-ttu-id="425ea-125">通常、アプリの一部に対するディープ リンクの設定には、サーバー上のルーティング ソリューションが必要です。</span><span class="sxs-lookup"><span data-stu-id="425ea-125">Deep linking to parts of the app typically requires a routing solution on the server.</span></span> <span data-ttu-id="425ea-126">そのルーティング ソリューションによって、アプリのルートに要求がリダイレクトされます。</span><span class="sxs-lookup"><span data-stu-id="425ea-126">The routing solution redirects requests to the root of the app.</span></span> <span data-ttu-id="425ea-127">たとえば、このリダイレクトは、IIS の URL 書き換え規則を使用して処理できます。</span><span class="sxs-lookup"><span data-stu-id="425ea-127">For example, this redirection can be handled using URL rewrite rules in IIS.</span></span>

<span data-ttu-id="425ea-128">Blazor とフルスタック .NET Web 開発のすべての利点を得るには、ASP.NET Core で Blazor WebAssembly アプリをホストします。</span><span class="sxs-lookup"><span data-stu-id="425ea-128">To get all the benefits of Blazor and full-stack .NET web development, host your Blazor WebAssembly app with ASP.NET Core.</span></span> <span data-ttu-id="425ea-129">クライアントとサーバーの両方で .NET を使用することにより、コードを簡単に共有し、言語、フレームワーク、ツールの 1 つの一貫したセットを使用してアプリを構築できます。</span><span class="sxs-lookup"><span data-stu-id="425ea-129">By using .NET on both the client and server, you can easily share code and build your app using one consistent set of languages, frameworks, and tools.</span></span> <span data-ttu-id="425ea-130">Blazor には、Blazor WebAssembly アプリと ASP.NET Core ホスト プロジェクトの両方を含むソリューションを設定するための便利なテンプレートが用意されています。</span><span class="sxs-lookup"><span data-stu-id="425ea-130">Blazor provides convenient templates for setting up a solution that contains both a Blazor WebAssembly app and an ASP.NET Core host project.</span></span> <span data-ttu-id="425ea-131">ソリューションをビルドすると、Blazor アプリからビルドされた静的ファイルは、フォールバック ルーティングが既に設定された状態で ASP.NET Core アプリによってホストされます。</span><span class="sxs-lookup"><span data-stu-id="425ea-131">When the solution is built, the built static files from the Blazor app are hosted by the ASP.NET Core app with fallback routing already setup.</span></span>

## <a name="blazor-server-apps"></a><span data-ttu-id="425ea-132">Blazor Server アプリ</span><span class="sxs-lookup"><span data-stu-id="425ea-132">Blazor Server apps</span></span>

<span data-ttu-id="425ea-133">[Blazor アーキテクチャ](architecture-comparison.md#blazor)に関する議論から、Blazor コンポーネントによってその出力が `RenderTree` と呼ばれる中間的な抽象化にレンダリングされることを思い出してください。</span><span class="sxs-lookup"><span data-stu-id="425ea-133">Recall from the [Blazor architecture](architecture-comparison.md#blazor) discussion that Blazor components render their output to an intermediate abstraction called a `RenderTree`.</span></span> <span data-ttu-id="425ea-134">次に、Blazor フレームワークによって、レンダリングされたものと以前にレンダリングされたものが比較されます。</span><span class="sxs-lookup"><span data-stu-id="425ea-134">The Blazor framework then compares what was rendered with what was previously rendered.</span></span> <span data-ttu-id="425ea-135">その相違点は DOM に適用されます。</span><span class="sxs-lookup"><span data-stu-id="425ea-135">The differences are applied to the DOM.</span></span> <span data-ttu-id="425ea-136">Blazor コンポーネントは、そのレンダリングされた出力の適用方法からは切り離されます。</span><span class="sxs-lookup"><span data-stu-id="425ea-136">Blazor components are decoupled from how their rendered output is applied.</span></span> <span data-ttu-id="425ea-137">そのため、コンポーネント自体は、UI を更新するプロセスと同じプロセスで実行する必要はありません。</span><span class="sxs-lookup"><span data-stu-id="425ea-137">Consequently, the components themselves don't have to run in the same process as the process updating the UI.</span></span> <span data-ttu-id="425ea-138">それどころか、同じコンピューター上で実行する必要もありません。</span><span class="sxs-lookup"><span data-stu-id="425ea-138">In fact, they don't even have to run on the same machine.</span></span>

<span data-ttu-id="425ea-139">Blazor Server アプリでは、コンポーネントはクライアント側のブラウザーではなく、サーバー上で実行されます。</span><span class="sxs-lookup"><span data-stu-id="425ea-139">In Blazor Server apps, the components run on the server instead of client-side in the browser.</span></span> <span data-ttu-id="425ea-140">ブラウザー内で発生した UI イベントは、リアルタイム接続を介してサーバーに送信されます。</span><span class="sxs-lookup"><span data-stu-id="425ea-140">UI events that occur in the browser are sent to the server over a real-time connection.</span></span> <span data-ttu-id="425ea-141">イベントは、正しいコンポーネント インスタンスにディスパッチされます。</span><span class="sxs-lookup"><span data-stu-id="425ea-141">The events are dispatched to the correct component instances.</span></span> <span data-ttu-id="425ea-142">コンポーネントはレンダリングされ、計算された UI の差分がシリアル化されてブラウザーに送信され、そこで DOM に適用されます。</span><span class="sxs-lookup"><span data-stu-id="425ea-142">The components render, and the calculated UI diff is serialized and sent to the browser where it's applied to the DOM.</span></span>

<span data-ttu-id="425ea-144">ASP.NET AJAX と <xref:System.Web.UI.UpdatePanel> コントロールを使用したことがある場合、Blazor Server のホスティング モデルにはなじみがあるかもしれません。</span><span class="sxs-lookup"><span data-stu-id="425ea-144">The Blazor Server hosting model may sound familiar if you've used ASP.NET AJAX and the <xref:System.Web.UI.UpdatePanel> control.</span></span> <span data-ttu-id="425ea-145">`UpdatePanel` コントロールにより、ページ上のイベントのトリガーに応じて、部分ページ更新の適用が処理されます。</span><span class="sxs-lookup"><span data-stu-id="425ea-145">The `UpdatePanel` control handles applying partial page updates in response to trigger events on the page.</span></span> <span data-ttu-id="425ea-146">トリガーされると、`UpdatePanel` によって部分的な更新が要求され、ページを更新する必要なくそれが適用されます。</span><span class="sxs-lookup"><span data-stu-id="425ea-146">When triggered, the `UpdatePanel` requests a partial update and then applies it without needing to refresh the page.</span></span> <span data-ttu-id="425ea-147">UI の状態は、`ViewState` を使用して管理されます。</span><span class="sxs-lookup"><span data-stu-id="425ea-147">The state of the UI is managed using `ViewState`.</span></span> <span data-ttu-id="425ea-148">Blazor Server アプリは、アプリがクライアントとのアクティブな接続を必要とするという点で若干異なります。</span><span class="sxs-lookup"><span data-stu-id="425ea-148">Blazor Server apps are slightly different in that the app requires an active connection with the client.</span></span> <span data-ttu-id="425ea-149">また、すべての UI の状態がサーバー上で保持されます。</span><span class="sxs-lookup"><span data-stu-id="425ea-149">Additionally, all UI state is maintained on the server.</span></span> <span data-ttu-id="425ea-150">これらの違いを除けば、2 つのモデルは概念的に似ています。</span><span class="sxs-lookup"><span data-stu-id="425ea-150">Aside from those differences, the two models are conceptually similar.</span></span>

## <a name="how-to-choose-the-right-blazor-hosting-model"></a><span data-ttu-id="425ea-151">適切な Blazor ホスティング モデルを選択する方法</span><span class="sxs-lookup"><span data-stu-id="425ea-151">How to choose the right Blazor hosting model</span></span>

- <span data-ttu-id="425ea-154">.NET サーバー側の依存関係がありません。</span><span class="sxs-lookup"><span data-stu-id="425ea-154">There's no .NET server-side dependency.</span></span> <span data-ttu-id="425ea-155">アプリはクライアントにダウンロードされた後、完全に機能します。</span><span class="sxs-lookup"><span data-stu-id="425ea-155">The app is fully functioning after downloaded to the client.</span></span>
- <span data-ttu-id="425ea-156">クライアントのリソースと機能が完全に活用されます。</span><span class="sxs-lookup"><span data-stu-id="425ea-156">Client resources and capabilities are fully leveraged.</span></span>

- <span data-ttu-id="425ea-169">サーバー上の .NET はアプリの実行に使用されるため、デバッグなどの既存の .NET ツールは想定どおりに動作します。</span><span class="sxs-lookup"><span data-stu-id="425ea-169">.NET on the server is used to run the app, so existing .NET tooling, such as debugging, works as expected.</span></span>
